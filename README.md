# Инструкция по запуску проекта

## Предварительные требования

- Docker и Docker Compose
- Go 1.25.0 или выше

## Шаг 1: Запуск зависимостей (Docker Compose)

Запустите все необходимые сервисы (PostgreSQL, Redis, MinIO, Adminer):

```bash
docker-compose up -d
```

Это запустит:
- **PostgreSQL** на порту `5432`
- **Redis** на порту `6379`
- **MinIO** на портах `9000` (API) и `9001` (консоль)
- **Adminer** (веб-интерфейс для БД) на порту `8081`

### Проверка работы сервисов

- MinIO Console: http://localhost:9001 (логин: `minioadmin`, пароль: `minioadmin`)
- Adminer: http://localhost:8081

## Шаг 2: Установка зависимостей Go

```bash
go mod download
```

## Шаг 3: Применение миграций базы данных

```bash
go run cmd/migrate/main.go
```

Это создаст все необходимые таблицы в базе данных.

## Шаг 4: Запуск приложения

```bash
go run cmd/awesomeProject/main.go
```

Приложение будет доступно на `http://localhost:8080`

## Альтернативный способ: сборка исполняемого файла

Если хотите собрать бинарный файл:

```bash
go build -o app.exe cmd/awesomeProject/main.go
./app.exe
```

## Остановка сервисов

Чтобы остановить все Docker-контейнеры:

```bash
docker-compose down
```

Чтобы остановить и удалить все данные (volumes):

```bash
docker-compose down -v
```

## Переменные окружения

Проект использует файл `app.env` для настроек подключения к базам данных. Убедитесь, что он содержит правильные значения (по умолчанию уже настроены для работы с docker-compose).

## Конфигурация

Основная конфигурация находится в `config/config.toml`. Там настраиваются:
- Порт сервера (по умолчанию 8080)
- Настройки MinIO
- Настройки JWT
- Настройки Redis (частично через переменные окружения)
